# 查重管理系统

## 🚀 极速启动（推荐）

为了方便使用，我已经为您创建了一个**一键启动程序**：`启动项目.exe`。

**使用方法：**
1. **直接双击** 根目录下的 `启动项目.exe`。
2. 程序会自动执行以下操作：
   - 检查并安装后端依赖 (`npm install`)
   - 检查并构建前端界面 (`npm run build`)
   - **自动设置开机自启**（以后电脑开机系统就会自动运行）
   - 启动后端服务
3. 启动完成后，浏览器访问 `http://localhost:3001` 即可使用。

*注意：第一次启动时因为需要安装插件和打包，可能需要 1-3 分钟，请耐心等待黑色窗口运行完成。*

---

## 📖 传统手动启动指南

1. 打开浏览器，访问 **https://nodejs.org**
2. 点击左边那个绿色大按钮 **「LTS」**（推荐版本，更稳定）下载安装包
3. 下载完成后双击 `.msi` 文件，一路点 **Next → Next → Install**，全部默认就行
4. 安装完成后**重启电脑**

验证是否安装成功：按 `Win + R`，输入 `cmd`，回车，在黑色窗口里输入：
```
node -v
```
如果显示类似 `v20.11.1` 这样的版本号，说明安装成功了。

---

## 第二步：解压项目文件

把下载的 `checklist-system.zip` 解压，放到一个好找的地方，比如：
```
C:\checklist-system
```

---

## 第三步：安装后端依赖

按 `Win + R`，输入 `cmd`，回车，然后依次输入以下命令（每行输完按一次回车）：

```bash
cd C:\checklist-system\checklist-system\backend
```
```bash
npm install
```

等待安装完成，看到光标重新出现就说明好了（大概需要1-3分钟，取决于网速）。

---

## 第四步：安装前端依赖并构建

继续在同一个黑色窗口里输入：

```bash
cd C:\checklist-system\checklist-system\frontend
```
```bash
npm install
```
```bash
npm run build
```

`npm run build` 这步会把前端页面打包，完成后会显示 `dist/` 字样，说明成功了。

---

## 第五步：启动系统

```bash
cd C:\checklist-system\checklist-system\backend
```
```bash
node server.js
```

看到下面这个画面说明启动成功了：

```
╔══════════════════════════════════════════╗
║        查重管理系统 已成功启动           ║
╠══════════════════════════════════════════╣
║  本机:   http://localhost:3001           ║
║  局域网: http://192.168.x.x:3001         ║
╚══════════════════════════════════════════╝
```

**注意：这个黑色窗口不能关，关了系统就停了。**

---

## 第六步：打开系统

打开浏览器（Chrome、Edge 都可以），地址栏输入：
```
http://localhost:3001
```

登录账号：
- 用户名：`admin`
- 密码：`admin123`

---

## 局域网其他电脑访问

启动成功后，黑色窗口里会显示一个 `192.168.x.x:3001` 的地址，**同一个 WiFi 下的其他电脑**，打开浏览器输入那个地址就能访问，不需要做任何额外设置。

---

## 以后每次开机启动

每次开机后，只需要打开 `cmd`，运行：
```bash
cd C:\checklist-system\checklist-system\backend
node server.js
```

## 常见问题

**问：`npm install` 卡很久没反应？**
换一个国内镜像源，输入这条命令再重试：
```bash
npm config set registry https://registry.npmmirror.com
```

**问：提示「不是内部或外部命令」？**
说明 Node.js 没安装好，或者安装后没有重启电脑，重启后再试。

**问：浏览器打开是空白或报错？**
检查第四步的 `npm run build` 有没有做，这步不能跳过。
